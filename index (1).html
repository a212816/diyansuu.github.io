Millions of people still go to bed hungry while food is being wasted every day. Surplusly uses technology to bridge that gap — helping communities share surplus food rather than discard it.

Learning Journey

Lab Work

Lab 1

UI Layout

Built the initial user interface for Surplusly using Jetpack Compose, establishing the visual foundation of the app.

In Lab 1, I built the initial UI layout for Surplusly using Jetpack Compose, focusing on clean screen structure and component placement. This was my first hands-on experience designing a mobile interface, and I learned how to organise composables effectively. It set the visual foundation for the entire app.
Lab 2

Interaction & State

Added click events, state management, and navigation between screens to make the app responsive to user input.

Lab 2 introduced user interaction by adding click events, state management, and navigation between screens in Surplusly. I learned how to handle user input and respond to it dynamically using Compose state. This made the app feel alive and responsive for the first time.
Lab 3

Material Design

Applied Material Design 3 components — colour schemes, typography, and cards — to give Surplusly a polished, consistent look.

In Lab 3, I applied Material Design 3 components to give Surplusly a polished and consistent look. I worked with colour schemes, typography, and card components to improve visual hierarchy. The app started to look professional and aligned with modern Android design standards.
Lab 4

Navigation

Implemented multi-screen navigation using Jetpack Navigation Compose, turning Surplusly into a fully navigable multi-page application.

Lab 4 focused on implementing multi-screen navigation using Jetpack Navigation Compose. I structured the app's navigation graph to allow smooth transitions between Home, Add Food, and Profile screens. This was a significant milestone as Surplusly became a fully navigable multi-page application.
Lab 5

Room Database

Integrated Room Database for local data persistence, allowing Surplusly to store and retrieve food listings offline.

Lab 5 introduced local data persistence using Room Database, allowing Surplusly to store and retrieve food listings offline. I implemented DAO interfaces, entities, and a repository pattern to manage data flow cleanly. This was the most technically challenging lab and gave me a solid understanding of local storage in Android.

Surplusly — Working Features

Food waste is a growing problem, yet surplus food often goes uneaten due to lack of visibility. Surplusly addresses this by providing a platform for users to list and discover surplus food in their community.

Building Project 1 taught me how to integrate multiple features into a coherent working app. I learned to manage navigation, UI state, and user flows together for the first time. It also highlighted areas for improvement that I carried into Project 2.

Project 2

Surplusly — Cloud + Barcode + API

Manually entering food details is tedious and error-prone, and data needs to persist across devices. Project 2 adds barcode scanning via CameraX and ML Kit, fetches product data from OpenFoodFacts API, and syncs everything to Firebase Firestore.

Project 2 was the most complex build I have undertaken, combining Room Database, Firebase Firestore, Retrofit, CameraX, and ML Kit barcode scanning in one app. I learned how to handle asynchronous API calls, manage cloud sync alongside local storage, and debug complex Gradle dependency conflicts. This project gave me real-world Android development experience that I am proud of.

Looking Back

Reflection & Learning Journey

Starting from Lab 1 where I built a simple UI, I have grown into a developer capable of building a full-stack Android app with cloud integration, external APIs, and camera hardware access. Each lab built on the previous one, and Surplusly evolved from a static screen into a dynamic, data-driven application.

The biggest challenge was managing multiple integrations in Project 2, but it taught me resilience, problem-solving, and the importance of clean architecture. Debugging Gradle dependency conflicts and synchronising Room with Firebase Firestore pushed me beyond my comfort zone and made me a better engineer.

I leave this course with both technical skills and a genuine passion for building apps that solve real-world problems — and the confidence that I can tackle complexity head-on.
